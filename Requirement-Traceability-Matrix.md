## Notes

- PASS in this matrix means “evidence-backed in this repository,” not “production load certified.”
- PARTIAL is used whenever the implementation is real but the runtime proof is incomplete, mock-based, or environment-dependent.
- NOT_VERIFIED is reserved for items that were not exercised at runtime in this workspace.
- Gap severity is assigned to the outstanding risk, not to the amount of code present.
